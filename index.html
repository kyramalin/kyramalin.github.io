While implementing the image classification it became obvious, that the classification results vary depending on the content and quality of the images. For example, the model may correctly classify clear images of most known categories like animals or vehicles. However, the model may struggle with less clear images, or those representing less common classes. Unexpectedly, there is also a limitation for some known categories (e.g. horse images). This can be due to the model's training data and its ability to recognize specific features in the images. Additionally, images with unusual angles, lighting, or backgrounds may challenge the model's accuracy. Overall, the main learning was about the potential and limitations of machine learning models in classifying images.

Frameworks & Technologies

  • ml5.js: In this code, ml5 is used to load and use the MobileNet model for image classification.
  • chart.js: Chart.js library is used to create the bar charts to display the classification results graphically in canvas elements.
  • JavaScript: With JavaScript interactivity and functionality is added, enabling actions like image classification using ml5.js, rendering bar charts with Chart.js, handling file uploads, and responding to user events like button clicks and file drops.
  • HTML: HTML provides the structure for the user interface, including elements such as buttons, divs, and canvases.
  • CSS: CSS takes care of the styling of the user interface (e.g. of the appearance of elements, providing visual feedback on user interactions).
